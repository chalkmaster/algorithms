-> Para definir o nível de complexidade das telas, foram dados pontos para cada elemento gráfico encontrado no sistema (cada componente gráfico, como label, caixa de entrada, etc, recebeu uma pontuação única).
Desta forma, a complexidade da tela pode ser obtida somando-se a complexidade de cada elemento que a compõe.
-> Para melhor caracterização do problema e facilidade de medição, os pontos referentes de acesso a dados foram agrupados nas telas ao qual pertencem (foram utilizados).
-> A complexidade de uma tela (já incluindo o custo de consumo de dados) é a soma de todos os elementos que ela utiliza.
-> As pontuações foram seguidas conforme definido abaixo:

Parametros de pontuação definidos:
	-> Campo de entrada simples: 1pt
	-> Campo de entrada com máscara de entrada: 2pts
	-> Campo de entrada com validação (tendo ou não máscara) : 3pt
	-> Campo de entrada com lookup: 10pts
	-> label: 1pt
	-> Acesso a Dados: 20 pts (pois contempla camada de acesso, configuração de sgbd, etc...)
	-> validação de regra de negócio ou segurança: 5pts
	-> exibição de dados de relatório: 30 pts (pois compreende paginações, N campos, etc...)
